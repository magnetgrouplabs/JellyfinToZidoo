   Note: These should NOT be in default SharedPreferences (they're in SecureStorage), but verify by removing them defensively. The password inclusion from SecureStorage is already correctly handled.

2. **Import: Auto-authenticate after settings restored** -- At the end of `importSettings()`, after all preferences are set and before the success toast, add auto-login:
   ```java
   // Auto-authenticate with restored credentials to get fresh token
   SharedPreferences prefs = PreferenceManager.getDefaultSharedPreferences(getApplicationContext());
   String serverUrl = prefs.getString("jellyfin_server_url", "");
   String username = prefs.getString("jellyfin_username", "");
   String password = SecureStorage.getInstance(getApplicationContext()).getString("jellyfin_password", "");

   if (!serverUrl.isEmpty() && !username.isEmpty() && !password.isEmpty()) {
       JellyfinApi.authenticate(serverUrl, username, password, getApplicationContext(),
           new JellyfinApi.SimpleCallback() {
               @Override
               public void onSuccess(String msg) {
                   runOnUiThread(() ->
                       Toast.makeText(getApplicationContext(),
                           "Settings imported and logged in successfully", Toast.LENGTH_LONG).show());
               }
               @Override
               public void onError(String error) {
                   runOnUiThread(() ->
                       Toast.makeText(getApplicationContext(),
                           "Settings imported. Login failed: " + error + " — please log in manually",
                           Toast.LENGTH_LONG).show());
               }
           });
   }
   ```
   Replace the existing success toast with this auto-login block. If credentials are missing/incomplete, show the original success toast without auto-login.
