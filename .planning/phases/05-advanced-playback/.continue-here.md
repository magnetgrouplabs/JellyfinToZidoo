### Bug fixes applied during this session (uncommitted):

1. **IntroSkipper API field names (CRITICAL FIX)**: `parseIntroSkipperResponse()` in JellyfinApi.java used wrong field names `IntroStart`/`IntroEnd` — real API uses `Start`/`End`. Also replaced `Valid` field check with `End > 0.0` null-safe check. Tests updated to match real API format. **This fix is working — intro skip now fires correctly.**

2. **stopZidoo() method added then REMOVED**: Added a `stopZidoo()` method using `Key.MediaStop` via Zidoo REST API (`/ZidooControlCenter/RemoteControl/sendkey?key=Key.MediaStop`). This was supposed to prevent Zidoo auto-advance at end of episode. **It broke the Up Next flow** — `Key.MediaStop` interferes with `finishActivity(98)` which is the Android lifecycle mechanism for cleanly closing the Zidoo player and triggering `onActivityResult`. The method has been removed.

3. **Credit skip and generic stop paths revised**: Both now use the same pattern: set `upNextTriggered = true` → `finishActivity(98)` → `onActivityResult` routes to `handleEpisodeCompleted()` → `launchUpNext()`. No `stopZidoo()` calls.
