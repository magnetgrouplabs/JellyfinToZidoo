<decisions_made>

- Progress poller cleanup moved from onStop to onDestroy — onStop fires when Zidoo player takes foreground, killing the poller before it can do anything
- Episode completion detection via video.path change in getPlayStatus poller — Zidoo auto-advances to next file in directory, we detect the path change
- After marking watched, open next episode detail page in Jellyfin client (not auto-play) — Jellyfin Android TV does NOT have a "Next Up" countdown for external players (confirmed via GitHub issues #5293, #3971, #3911), so we open the next episode page instead
- handleEpisodeCompleted wrapped in runOnUiThread — poller runs on background executor, but startActivity/finish must run on UI thread
- Removed relaunchCallerOrFinish entirely — getLaunchIntentForPackage always opened Jellyfin home screen, moveTaskToFront didn't help either. Just finish() for non-watched, searchNextEpisode() for watched.
