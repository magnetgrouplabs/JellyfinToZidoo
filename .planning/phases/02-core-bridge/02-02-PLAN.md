  <name>Task 2: Wire SettingsActivity to handle API key via EncryptedSharedPreferences</name>
  <files>app/src/main/java/com/jellyfintozidoo/SettingsActivity.java</files>
  <action>
    Modify SettingsActivity.java to handle the API key preference specially:

    1. Add imports for SecureStorage (from Plan 01) and android.os.Handler, android.os.Looper.

    2. In showRootSettings(), after setPreferencesFromResource, add API key preference handling:
       - Find "jellyfin_api_key" EditTextPreference
       - On preference load: read actual value from SecureStorage.getInstance(context), set the preference's text to it, and set summary to "********" if non-empty or "Not set" if empty
       - Set OnPreferenceChangeListener: when value changes, write new value to SecureStorage.getInstance(context).edit().putString("jellyfin_api_key", newValue).apply(), update summary to "********" or "Not set", and return false (prevent writing to default SharedPreferences)
       - Set OnBindEditTextListener to mask input: editText.setInputType(InputType.TYPE_CLASS_TEXT | InputType.TYPE_TEXT_VARIATION_PASSWORD)

    3. Add Test Connection button handler:
       - Find "jellyfin_test_connection" Preference
       - Set OnPreferenceClickListener:
         * Read server URL from default SharedPreferences ("jellyfin_server_url")
         * Read API key from SecureStorage.getInstance(context)
         * If either is empty, show Toast "Please configure Server URL and API Key first"
         * Otherwise call JellyfinApi.testConnection(serverUrl, apiKey, new SimpleCallback() { onSuccess -> Toast success message, onError -> Toast error message })

    4. Update exportSettings() to also include the server URL (it's in regular SharedPreferences so it exports automatically). For the API key: do NOT export it (same pattern as smbPassword -- filter it out). Add "jellyfin_api_key" to the export filter:
       `.filter(s -> !s.getKey().startsWith("smbPassword") && !s.getKey().equals("jellyfin_api_key"))`

    5. Update importSettings() to handle the server URL (it imports automatically via EditTextPreference). The API key won't be in the import file (since we don't export it), so no special handling needed.

    6. Strip trailing slash from server URL on preference change: add an OnPreferenceChangeListener on "jellyfin_server_url" that strips trailing "/" from the value before saving.
  </action>
