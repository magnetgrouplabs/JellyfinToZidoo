  <what-built>Complete Phase 2 bridge: Jellyfin settings, API key auth, intent interception, path substitution, Zidoo player launch with resume position, debug screen with full pipeline trace.</what-built>
  <how-to-verify>
    1. **Settings screen** -- Open JellyfinToZidoo app on Zidoo device:
       - Verify "Jellyfin Server" section appears at the TOP
       - Enter your Jellyfin server URL (e.g., http://192.168.x.x:8096)
       - Enter your API key (generated from Jellyfin admin dashboard)
       - Tap "Test Connection" -- should show success toast
       - Navigate settings with D-pad -- all sections should be reachable
       - Verify existing settings (debug toggle, player, substitution) still work

    2. **Debug screen** -- Enable debug mode in settings, then:
       - From Moonfin or Jellyfin Android TV client, play a movie that you've partially watched
       - JellyfinToZidoo should intercept and show the debug screen
       - Verify it shows: Jellyfin Item ID (UUID), API Path (server file path), Path Substitution (SMB path), View Offset (resume position in ms)
       - Press Play button on debug screen

    3. **Playback** -- After pressing Play:
       - Zidoo native player should launch with the correct file
       - Playback should resume at the stored position (not from start)
       - If you haven't set up path substitution, you'll see "ERROR: No substitution found" which is expected

    4. **Error handling** -- Test edge cases:
       - Clear the API key, try playing -- should show "Jellyfin server not configured" error
       - Enter wrong API key, try playing -- should show API error on debug screen
  </how-to-verify>
  <resume-signal>Type "approved" if the bridge works end-to-end, or describe any issues you see.</resume-signal>
