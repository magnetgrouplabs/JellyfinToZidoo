- **`finishActivity(98)` chosen as stop method**: Since we launched Zidoo with `startActivityForResult(_, 98)`, this should tell Android to finish that specific activity. Not yet tested.

## Flow When Working
1. Poller polls Zidoo getPlayStatus every 10s (3s when <60s remaining)
2. At 30s before end: set `upNextTriggered=true`, call `finishActivity(98)`
3. Zidoo player exits → `onActivityResult(98, ...)` fires
4. `stopProgressPoller()`, sees `upNextTriggered=true` → `handleEpisodeCompleted()`
5. `handleEpisodeCompleted`: `handlingPlaybackResult=true`, reportPlaybackStopped → markAsWatched → `launchUpNext()`
6. `launchUpNext`: `waitingForUpNext=true`, getNextUpWithDetails → start UpNextActivity
7. User clicks Play Now → `onActivityResult(UP_NEXT_REQUEST_CODE, OK)` → doSubstitution(nextServerPath) → buildZidooIntent → startActivityForResult
8. User clicks Cancel → `onActivityResult(UP_NEXT_REQUEST_CODE, CANCELED)` → finish()
