<decisions_made>

- Option B chosen for stopping Zidoo: User explicitly rejected Option A (Up Next on top of already-playing episode), Option C (self-sufficient UpNextActivity), and Option D (no Up Next on natural completion). User wants the player STOPPED before Up Next appears — like Netflix.
- finishActivity(98) does NOT work: It kills Play itself (isFinishing=true), not just the Zidoo player. Android forces FLAG_ACTIVITY_NEW_TASK, breaking the result contract. Log evidence: startActivity called from non-Activity context; forcing Intent.FLAG_ACTIVITY_NEW_TASK
- OkHttp callbacks run on background threads: All finish()/startActivityForResult() calls from JellyfinApi callbacks MUST be wrapped in runOnUiThread(). This was the root cause of the original crash (Bug 2).
- Reverse substitution fails for user paths: Log showed Reverse substitution failed for /data/system/smb/192.168.0.154... — the SMB path format from Zidoo does not match substitution rules. This is a separate issue for later.
- Up Next styling: Bottom-aligned layout, Jellyfin purple (#AA5CC3) Play Now button, ghost Cancel button, "Playing in X" countdown text, fullscreen theme.
</decisions_made>

<context>
The Zidoo Z9X Pro has a local HTTP API on port 9529 that the poller already uses for getPlayStatus. The stop endpoint likely exists at /ZidooVideoPlay/stopPlay or similar. The OkHttpClient singleton (getLocalClient()) with 3-second timeouts is already set up for this.

The flow should be:
1. Poller detects currentPositionMs >= 90% of lastKnownDurationMs
2. Set upNextTriggered = true
3. Call http://127.0.0.1:9529/ZidooVideoPlay/stopPlay via getLocalClient()
4. Zidoo player stops and returns result to Play.onActivityResult(98, ...)
5. onActivityResult sees upNextTriggered=true then handleEpisodeCompleted()
6. handleEpisodeCompleted: reportPlaybackStopped then markAsWatched then launchUpNext()
7. launchUpNext: getNextUpWithDetails then start UpNextActivity (Play is NOT finishing, clean screen)
8. User presses Play Now then onActivityResult(UP_NEXT_REQUEST_CODE, OK) then resolve path then launch Zidoo
9. User presses Cancel then onActivityResult(UP_NEXT_REQUEST_CODE, CANCELED) then finish()
